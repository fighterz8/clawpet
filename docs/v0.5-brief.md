# Clawpet v0.5 Brief

## Priority order

0. **Native packaged app onboarding**
   - Normal users should download/open a `.exe` or `.dmg`, not install Node/git/Rust.
   - The Tauri app should own the setup surface: runtime status, demo, pair code, and OpenClaw handoff.
   - Pairing should be one-time for normal use: desktop runtime persists its token locally, OpenClaw persists the matching token, and reopening the app should reconnect without a fresh pair code unless the token is rotated or local app data is cleared.
   - CLI/wizard remains for development and OpenClaw-side automation, not the primary user experience.

1. **Make behavior feel alive and accurate**
   - First user prompt should trigger immediately: `thinking` + `Reading your prompt…`.
   - Tool bubbles should describe the actual current process: reading files, running commands, searching web, generating image, waiting for input, done.
   - Prefer accurate/statusful copy over random cute flavor.

2. **Simplify install + demo**
   - README Quickstart should show the 60-second same-machine path first.
   - Move Tailscale/pair/daemon details to Cross-machine setup.
   - `CLAWPET_DEMO=1` should cycle states every 6s without OpenClaw running.

3. **README marketing pass**
   - Hero GIF above the fold: idle → thinking → focused → happy.
   - One-liner: “A pixel pet that watches your AI assistant work.”
   - Badges: license, Tauri, Node, build status.
   - Architecture diagram collapsed in `<details>` with deeper docs linked.
   - Support section + in-action GIFs near bottom.

4. **Avatar overhaul / v0.5 animation**
   - Add backward-compatible frame support to bundles.
   - Single-PNG bundles remain valid as 1-frame loops.
   - Dawn-v0 and Pip-v0 must continue loading unchanged.
   - Dawn-v1 should ship with animated per-state loops.

## Avatar generation cost caveat

Avatar overhaul is likely the most expensive part of polish. Generating/editing coherent multi-state, multi-frame pixel-art characters may require many image-generation attempts. User-facing docs should say avatar generation can incur non-trivial model/API costs, especially when iterating for consistency.

## Frame budget target

| State | Frames | Behavior |
| --- | ---: | --- |
| idle | 6 | blink + gentle bob |
| sleepy | 8 | slow breath |
| thinking | 4 | loop |
| focused | 4 | subtle loop |
| happy | 4 | bounce |
| alert | 2 | flash |
