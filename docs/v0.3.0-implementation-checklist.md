# Clawpals v0.3.0 Implementation Checklist

A file-by-file checklist for the `v0.3.0` expression-priority release.

## Core runtime model

### `src/contracts/avatarEvent.ts`
- [ ] Preserve backward compatibility for existing `avatar.state` payloads
- [ ] Document optional metadata fields used by runtime arbitration:
  - [ ] `sourceClass`
  - [ ] `lingerMs`
  - [ ] `sessionKey`
  - [ ] `channel`
  - [ ] `surface`
  - [ ] `replacePolicy` (optional/future-safe)
- [ ] Keep validation permissive enough that older emitters still work

### `src/runtime/stateStore.ts`
- [ ] Introduce runtime event source classification
- [ ] Add event-priority mapping:
  - [ ] `user-requested`
  - [ ] `OpenClaw expression`
  - [ ] `system signal`
- [ ] Add linger resolution per class
- [ ] Add active foreground tracking
- [ ] Add arbitration outcomes:
  - [ ] `shown`
  - [ ] `replaced`
  - [ ] `suppressed`
  - [ ] `skipped`
- [ ] Add expression cooldown / duplicate suppression
- [ ] Keep one foreground avatar slot only
- [ ] Preserve idle/sleepy fallback behavior when no foreground event remains active
- [ ] Record arbitration reasons for log/UI visibility

### `src/runtime/app.ts`
- [ ] Keep `/avatar/state` input contract backward compatible
- [ ] Return arbitration metadata in POST responses when useful
- [ ] Keep `/events` returning the richer runtime log entries
- [ ] Avoid breaking existing runtime consumers that only read `status.avatar.state` / `status.avatar.bubble`

### `src/runtime/server.ts`
- [ ] Keep demo/runtime bootstrap working with the updated event model
- [ ] Optionally seed demo metadata for source class testing

## Emitters / source annotation

### `skills/clawpals/bin/clawpals.mjs`
- [ ] Add explicit metadata to sends/reacts so runtime does not have to infer everything from display names
- [ ] Mark direct manual sends as either `system signal` or `user-requested` intentionally
- [ ] Mark non-daemon `react` emits as `OpenClaw expression`
- [ ] Set default linger hints where useful

### `skills/clawpals/bin/clawpals-daemon.mjs`
- [ ] Mark daemon events explicitly as `system signal`
- [ ] Add provenance/session metadata later when available
- [ ] Keep zero-token daemon phrase system intact

## Activity log UI

### `src/onboard.tsx`
- [ ] Read richer event-log entry fields from `/events`
- [ ] Show arbitration outcomes in the Activity Log
- [ ] Show clear source labels from runtime metadata instead of heuristic-only parsing where possible
- [ ] Preserve the current compact style language
- [ ] Keep Activity Log as the authoritative user-facing truth surface

### `src/onboard.css`
- [ ] Add compact visual styles for outcome labels / arbitration notes
- [ ] Ensure suppressed/skipped rows remain readable without making the log noisy
- [ ] Preserve the current compact/clipped-safe tab layout
