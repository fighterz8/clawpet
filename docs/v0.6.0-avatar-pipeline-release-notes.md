# Clawpals v0.6.0 — Avatar Pipeline Reliability Release Notes

Status: planned release notes / implementation branch notes

## Recommended version

Use **v0.6.0** for this avatar-pipeline reliability release.

Why minor version instead of patch:

- Adds new agent-facing avatar workflow commands.
- Adds a versioned pipeline job contract.
- Changes build behavior with `preserve-canvas` and `anchor-locked` registration modes.
- Adds deterministic animation, provider abstraction, targeted repair, vision QA, and sprite-sheet experiment support.
- Makes avatar generation substantially more automated, even though real provider-backed art generation is still future work.

Keep generated avatar bundle versions separate from the app/package version. For example:

- `0.2.0-local-animated` — legacy/local animated bundles.
- `0.2.0-mock-generated` — mock-provider pipeline fixture.
- `0.3.0-generated` — future regenerated avatars from the reliable pipeline.
- `0.3.0-openai-preview` / `0.3.0-gemini-preview` — future provider-specific preview bundles.

## Highlights

### Stable/reliable baseline

- Added explicit avatar pipeline job contract fields:
  - `pipelineJobSchemaVersion`
  - `generation.strategy`
  - `generation.sourceImageContract`
  - exact `generation.locked.paletteHex`
  - exact `generation.locked.outlineHex`
  - `registration`
- Added builder registration modes:
  - `preserve-canvas`
  - `anchor-locked`
  - `legacy-crop` compatibility mode
- Added deterministic animation generation:
  - six state anchors can produce full frame loops without generated frame deltas
- Added post-build QA artifacts:
  - `post-build-coherency-report.generated.json`
  - `repair-queue.generated.json`
  - `contact-sheet.generated.png`
- Added targeted repair flow:
  - repair only queued failed frames
  - keep passing frames untouched
  - provider-backed repair hook available
- Added mock provider:
  - offline-safe fixture generation
  - validates pipeline mechanics without real image API calls
- Added vision QA gate:
  - provider-backed structured review of contact sheet
  - mock provider supports local/CI path
- Added skill-facing command wrappers:
  - `clawpals avatar validate`
  - `clawpals avatar generate`
  - `clawpals avatar slice-sheet`
  - `clawpals avatar animate`
  - `clawpals avatar repair`
  - `clawpals avatar build`
  - `clawpals avatar vision-qa`
  - `clawpals avatar verify`

### Experimental

- Sprite-sheet slicing is opt-in only.
- Use it for experiments/comparisons, not as the default reliable path.

### Not yet implemented

Real image provider integration is intentionally stubbed. Preferred future provider targets:

- OpenAI `gpt-image-2`
- Gemini `gemini-3.1-flash-image-preview`

The provider interface is ready, but real calls for identity anchors, state anchors, reference edits, and semantic repair still need implementation.

## Validated test path

The mock full-pipeline test passed:

```bash
npm run avatar:pipeline -- generate jobs/pocket-golem-mock.sample.json
npm run avatar:pipeline -- animate .avatar-pipeline/pocket-golem-mock-v0/generated-job.generated.json
npm run avatar:pipeline -- build .avatar-pipeline/pocket-golem-mock-v0/animated-job.generated.json
npm run avatar:pipeline -- vision-qa .avatar-pipeline/pocket-golem-mock-v0/animated-job.generated.json
clawpals avatar push /home/nick/.openclaw/workspace/local_avatars/pocket-golem-mock-v0/bundle
clawpals status
```

Validated behavior:

- mock anchors generated
- deterministic frames generated
- bundle built
- post-build QA passed
- contact sheet generated
- vision QA mock report passed
- runtime push/select worked
- state cycling worked

## Known limitations

- Mock provider art is intentionally static/simple; it validates plumbing, not final visual quality.
- Existing Pocket Golem remains a validation fixture, not the final regenerated avatar.
- Existing golem feedback to fix later:
  - green background leakage
  - cross-state anchor drift
  - too-static animation in mock/provider test art
- Real provider generation/repair is required before regenerating Pocket Golem or creating polished new avatars.

## Recommended release summary

Clawpals v0.6.0 makes custom avatars agent-buildable: OpenClaw can scaffold jobs, generate or collect anchors, animate deterministically, run QA, repair failed frames, build review artifacts, push bundles to the paired runtime, and verify the selected avatar — all without manual desktop asset editing.
